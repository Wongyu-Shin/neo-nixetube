# Agent-Harness 아키텍처 카탈로그 — Loop Report

## TL;DR

13 iteration 루프로 HyperAgent·Reflexion·Voyager·SWE-agent·ADAS 외 6개
방법론을 읽어 **13개의 rippable harness feature**로 증류. axis1(inner/outer)
× axis2(pre/in/post-loop) 2×3 그리드의 **모든 셀이 최소 1개 이상으로 채워짐**.
각 feature는 (a) 프론트매터 schema (b) 논문/코드 citation (c) 재현 가능한
bash TC (d) 양적 rippable 신호를 포함. 루프 중 TC가 스스로 **2건의 citation
URL 포맷 불일치를 적발**하여 rework → pass로 수렴, schema 강제력이 실제로
작동함을 실증.

## 각 feature 한 줄 요약 (iteration 순)

1. **reflexion** (outer, in-loop) — Shinn 2023. 실패한 iteration의 "왜" 를 자연어로 영속화. git은 *무엇*을, reflexion은 *왜*를 기억.
2. **voyager-skill-library** (outer, pre-loop) — Wang 2023. 성공한 iteration을 skill 카드로 저장, 다음 Goal들이 retrieve해 재활용.
3. **swe-agent-aci** (inner, pre-loop) — Yang 2024. CC의 tool surface/permission/hook을 프로젝트 맞게 curation. "모델은 그대로, ACI만 바꿔도 3.8→18%".
4. **hyperagent-planner-routing** (outer, pre-loop) — Phan 2024. Planner/Navigator/Editor/Executor 역할 분할, Planner만 HITL 채널 소유 — axis-2 헌장의 구조적 실현.
5. **adas-meta-agent-search** (outer, pre-loop) — Hu 2024. 이 카탈로그 자체가 ADAS 스텝의 집합. archive-conditioned 제안 + novelty filter.
6. **llm-as-judge-audit** (outer, post-loop) — Zheng 2023. 루프를 돌린 모델과 *다른* 모델로 심판, self-enhancement bias 10% 차단. 프로젝트 기존 GAN 노이즈 경험과 직접 연결.
7. **cc-hook-guardrail** (inner, in-loop) — CC 공식 hooks. PreToolUse deny 목록으로 `rm -rf`/`git push --force` 등 비상정지. 모델이 작을수록 부담 큼.
8. **cc-post-loop-slash** (inner, post-loop) — CC slash command. 루프 종료시 템플릿 기반 MDX 리포트 생성 — **지금 읽고 있는 이 문서가 그 실체**.
9. **plateau-detection** (outer, in-loop) — Prechelt 2012 + Karpathy patience. 노이즈 하에서 "5번 연속 discard" 룰이 오작동하는 걸 patience + trend-slope CI로 보정.
10. **statistical-tc-runner** (outer, in-loop) — Welch t-test + MLPerf 패턴. axis-1 메타 feature: 모든 다른 feature의 TC가 smoke → 통계 tier로 올라탈 인프라.
11. **harness-rip-test** (outer, in-loop) — 프로젝트 기존 `scripts/test-harness-rip.sh` 를 catalog 엔트리로 승격. 모든 feature가 with/without 2-arm rip test 동봉 의무.
12. **noise-aware-ratchet** (outer, in-loop) — Chen 2016 + 프로젝트 기존 "ratchet MAX" 교훈. 3 규칙: (a) best-ever anchor (b) 주기적 재측정 (c) σ 마진 초과시만 keep.
13. **plan-mode-discipline** (inner, pre-loop) — CC Plan mode + ExitPlanMode. HITL 채널 단일화 + Stop-hook 트랜스크립트 린터로 in-loop HITL leak 0/goal 강제.

각 feature의 전문은 `harness/features/<name>.md` (spec) +
`harness/research/<name>.md` (논문 distillation) + `harness/tests/<name>.sh`
(TC) 3종 세트로 영속화.

## 루프 자체가 확인한 설계 원칙 두 개

### 1. Schema-as-guard가 실제로 회귀를 잡는다

iter 6 (`llm-as-judge-audit`) 와 iter 12 (`noise-aware-ratchet`) 에서, 새 feature
의 citation 포맷이 기존 카탈로그 관례 (`arxiv.org/abs/<id>` 풀 URL) 와 어긋난
상태에서 commit → verify 순간 TC가 즉시 잡아냄. 사람이 발견해야 했을 표기
일관성 오류가 자동 감지된 것. **이는 axis-1 요구사항 "TC는 자동으로 트리거
되어야 함"의 가장 단순한 구현이 이미 구현되어 작동하고 있음**을 증명.

### 2. 동일 헌장을 axis로 이원화하는 설계가 rippable 안전망이 된다

`hyperagent-planner-routing` (outer) 과 `plan-mode-discipline` (inner) 은 모두
"HITL을 loop 밖으로" 헌장을 실현하지만 구현층이 다름. CC가 한쪽을 네이티브로
흡수해도 (가령 Plan mode가 강제성을 내재화) 다른 쪽이 남아 커버. 반대도 성립.
이는 axis-1이 단순 분류가 아니라 **설계 보험**으로 기능한다는 경험적 근거.

## 알려진 한계 / 다음 루프의 후보

현재 13개 TC 전부 **smoke tier** (존재성 + 구조 grep). axis-1 헌장이 요구

회귀를 잡고 있다는 사실은 위 §1에서 증명됨.

공개 harness라는 점에서 catalog 엔트리 가치 최상위.
